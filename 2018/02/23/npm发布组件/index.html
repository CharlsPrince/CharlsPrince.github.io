版本号规范

常规组件发布版本都是使用语义化版本，npm社区版本号规则采用的是semver（语义化版本），版本格式为:主版本号.次版本号.修订号,版本号递增规则如下：

  1. 主版本号：当你做了不兼容的 API 修改，
  2. 次版本号：当你做了向下兼容的功能性新增，
  3. 修订号：当你做了向下兼容的问题修正。

先行版本号及版本编译信息可以加到“主版本号.次版本号.修订号”的后面，作为延伸。
